part1:



git init → ایجاد یک مخزن (repository) جدید



git clone <repo> → کلون کردن یک مخزن از GitHub



git status → مشاهده وضعیت فایل‌ها و تغییرات



git add <file> → اضافه کردن فایل‌ها به staging area



git commit -m "<message>" → ذخیره تغییرات با یک پیام توضیحی



git log → مشاهده تاریخچه commit‌ها



git log --oneline → مشاهده تاریخچه commit‌ها به صورت خلاصه



git diff → مقایسه تغییرات بین نسخه‌ها



git checkout <branch> → جابجایی بین شاخه‌ها



git branch → مشاهده و مدیریت شاخه‌ها



git merge <branch> → ادغام تغییرات یک شاخه به شاخه‌ی جاری



git fetch → دریافت تغییرات از remote بدون ادغام



git pull → دریافت و ادغام تغییرات از remote



git push → ارسال تغییرات محلی به remote



git rm --cached <file> → حذف فایل از tracking بدون حذف فیزیکی



ایجاد و ویرایش .gitignore → نادیده گرفتن فایل‌ها یا پوشه‌های خاص



ایجاد feature branch و مدیریت آن



ایجاد pull request و merge کردن آن در GitHub



part2:



مدیریت فایل‌های حساس و اشتباهی:



اضافه شدن فایل password.txt به repository و نیاز به حذف آن از Git بدون پاک کردن فایل از سیستم.



هماهنگ کردن شاخه‌های محلی و remote:



مشکل در پیاده‌سازی merge بین شاخه‌های feature/add-skills و main.



درک تفاوت fetch و pull:



fetch فقط تغییرات را دریافت می‌کند، بدون ادغام.



pull تغییرات را دریافت و به شاخه جاری ادغام می‌کند.



ایجاد و مدیریت pull request در GitHub:



فهم اینکه base branch و compare branch چه هستند و چگونه تغییرات را در پروژه اصلی merge کنیم.



part3:



اهمیت commit‌های معنادار و منظم:



هر commit باید دقیقاً تغییرات انجام شده را توضیح دهد تا تاریخچه واضح باشد.



استفاده از .gitignore برای فایل‌های حساس یا موقت:



جلوگیری از اضافه شدن فایل‌های غیرضروری مثل password.txt و فایل‌های log.



کار با شاخه‌ها و pull request‌ها:



یاد گرفتید که چگونه شاخه‌های feature بسازید، تغییرات را تست کنید و با pull request در پروژه ادغام کنید.
